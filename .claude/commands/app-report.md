# 명리연구소 앱 현황 리포트

앱인토스 콘솔(apps-in-toss.toss.im)에 Playwright로 접속하여 명리연구소 앱의 주요 지표를 수집하고 분석 리포트를 생성합니다.

## 수집 항목

아래 데이터를 앱인토스 콘솔에서 Playwright MCP를 사용하여 수집하세요:

### 1. 대시보드 (홈)
- 앱인토스 콘솔 접속: apps-in-toss.toss.im
- 명리연구소 앱 선택
- 대시보드에서 DAU, WAU, MAU 등 유입자 수 확인
- 스크린샷 캡처

### 2. 분석 > 유저 지표
- 일별/주별 방문자 수 추이
- 신규 vs 재방문 비율
- 리텐션(D1, D7, D30)
- 체류 시간 (평균/중간값)
- 스크린샷 캡처

### 3. 분석 > 전환 지표
- 등록해둔 전환 이벤트 성과 (운세결과확인 impression, 체류시간 1분, D7 재방문)
- 광고 시청 수 (리워드 광고)
- 스크린샷 캡처

### 4. 분석 > 퍼널/이벤트
- 기능별 사용 비율 (사주풀이/꿈해몽/띠별운세/궁합)
- 각 기능의 완료율 (입력→결과 도달 비율)
- 이탈 구간 파악
- 스크린샷 캡처

### 5. 스마트 발송
- 발송 캠페인별 CTR (클릭률)
- 발송 수, 도달 수, 클릭 수
- 캠페인별 성과 비교
- 스크린샷 캡처

### 6. 프로모션
- 포인트 지급 현황
- 참여자 수
- 예산 소진율
- 스크린샷 캡처

### 7. 리뷰
- 최근 리뷰 내용 및 평점
- 총 리뷰 수
- 부정 리뷰 있으면 원문 포함
- 스크린샷 캡처

### 8. 공유 리워드
- 공유 바이럴(contactsViral) 참여 현황
- 황금 열람권 지급 수
- 공유→유입 전환율 (가능하면)
- 스크린샷 캡처

### 9. 유저 세그먼트 분석
- 성별/연령대별 분포
- 시간대별 접속 패턴 (피크타임 파악)
- 세그먼트별 리텐션 차이
- 스크린샷 캡처

### 10. 앱 노출/검색
- 앱인토스 내 노출 순위 (카테고리 내 위치)
- 검색 유입 키워드 (있으면)
- 스크린샷 캡처

### 11. 공지사항 확인
- 앱인토스 콘솔의 공지사항/알림 메뉴 확인
- 새로운 정책 변경, 기능 업데이트, 가이드라인 변경 등 확인
- 우리 앱에 영향이 있는 공지사항이 있으면 원문 요약 + 필요한 대응 사항 정리
- 심사 기준 변경, 광고 정책 변경, 새로운 SDK 버전 등 중요 업데이트 체크
- 프로모션/이벤트 관련 공지 (신규 프로모션 타입, 할인 등)도 확인

## 수집 방법

1. Playwright MCP `browser_navigate`로 앱인토스 콘솔 접속
2. 각 메뉴를 순회하며 `browser_snapshot` + `browser_take_screenshot`으로 데이터 캡처
3. 페이지에서 숫자 데이터를 읽어서 정리
4. 탭/드롭다운이 있으면 기간별(오늘/7일/30일) 데이터 모두 확인

## 주의사항
- Chrome Default 프로필(경민/sinjin3400@gmail.com)로 접속
- 콘솔 로딩이 느릴 수 있으니 각 페이지 이동 후 충분히 대기 (browser_wait_for 활용)
- 데이터가 없는 항목은 "데이터 없음"으로 표기하되, 왜 없는지 추정 코멘트 추가
- 숫자는 가능한 정확하게 읽되, 읽기 어려우면 스크린샷으로 대체
- 리포트는 텔레그램 메시지 글자수 제한(4096자) 고려해서 필요시 2개 메시지로 분할
- 스크린샷은 핵심 지표가 담긴 것 2-3장을 텔레그램 첨부로 같이 전송
